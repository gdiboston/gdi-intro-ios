Overview of Apple's Navigation Guidelines

  • Users should always know where they are in your app and how to get to their next destination
  • Use a navigation bar to give users an easy way to traverse a hierarchy of data
  • Use a tab bar to display several peer categories of content or functionality
  • In general, itâ€™s best to give users one path to each screen

Navigation Controller

The views of a navigation controller

What does the navigation controller do?

  • Respond to user actions by pushing new content view controllers onto the stack or popping content view controllers off of the stack
  • Monitor changes to the Navigation Stack

Push a view controller

Push a view controller

             UIViewController *viewControllerToPush = [UIViewController alloc] init];

[self.navigationController pushViewController:viewControllerToPush animated:YES];

Pop back to the previous view controller or the root view controller

            [self.navigationController popViewControllerAnimated:YES];

or

[self.navigationController popToRootViewControllerAnimated:YES];

Present a view controller

            UIViewController *viewControllerToPresent = [UIViewController alloc] init];

[self.navigationController presentViewController:viewControllerToPresent animated:YES completion:nil];

Dismiss the presented view controller

            [previousViewController dismissViewControllerAnimated:YES completion:nil];
